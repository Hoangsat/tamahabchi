## 1. КРАТКОЕ ОПИСАНИЕ (ВИДЕНИЕ)

**Tamahabchi** — это мобильная игра, которая помогает игроку развивать реальные навыки и формировать полезные привычки через игровую механику.

**Главный цикл:**
Игрок выбирает навык → запускает режим фокуса (таймер) → получает награду (монеты + опыт питомцу + % к навыку) → навык растёт → паутинная диаграмма меняется → питомец сыт и счастлив → открывается новый контент

**Ключевая фишка:**  
Бесконечная паутинная диаграмма (radar chart), куда игрок может добавлять ЛЮБОЕ КОЛИЧЕСТВО НАВЫКОВ (3, 10, 20, 100 — без ограничений). Каждый навык — отдельная ось. Длина оси = процент развития (0–100%). Диаграмма показывает ТОП-12 самых развитых навыков.

**Вторая ключевая фишка — умные боссы:**  
Боссы имеют свою паутину навыков. Если твой навык ниже, чем у босса — ты получаешь урон. Если выше — наносишь больше урона. Боссы показывают твои слабые места и мотивируют развивать паутину равномерно.

**Третья ключевая фишка — два типа ежедневных заданий:**
- Навыковые задания — привязаны к паутине («30 мин математика»)
- Простые рутины — забота о себе («выпить воду», «8 часов сна», «принять душ»)

**Эмоциональное обещание:**  
«Ты развиваешь себя в реальной жизни — твой питомец и паутина растут вместе с тобой. А боссы подсказывают, куда двигаться дальше».

## 2. ГЛОБАЛЬНАЯ АРХИТЕКТУРА

### 2.1 Слои (строгие правила — не нарушать)

| Слой | Компоненты | Разрешено | ЗАПРЕЩЕНО |
|------|------------|-----------|------------|
| Bootstrap | GameBootstrap, SceneLoader | Загружать сцены, инициализировать менеджеры | Любую UI-логику, доступ к трансформам |
| Core | GameManager (только связывает) | Давать доступ к сервисам и runtime state | Хранить данные игрока, считать баланс |
| Systems | PetSystem, SkillSystem, MissionSystem, BattleSystem, ShopSystem | Содержать игровую логику, менять PlayerState | Обращаться к UI (кнопкам, текстам) |
| Data | PlayerData, SaveManager, Definitions | Загружать/сохранять JSON, хранить конфиги | Содержать игровую логику |
| UI | Все Panel, Button, Text, Slider | Показывать данные, отправлять команды в системы, слушать события | Считать награды, менять save напрямую |
| Events | EventBus (C# events / UnityEvents) | Передавать события между системами | Прямые ссылки между системами |

**Текущая runtime-реальность:**
- `Systems` уже являются source of truth для навыков, питомца, миссий и магазина.
- `GameManager` пока ещё толще целевого варианта и держит часть orchestration/UI glue.
- `Bootstrap`, `SceneLoader` и полноценный `EventBus` остаются целевым архитектурным направлением.
- Ниже фиксируется target structure, а не точная текущая карта файлов один-в-один.

### 2.3 EventBus (примеры событий)

```csharp
// Событие изменения навыка
public class SkillProgressEvent {
    public string skillId;
    public string skillName;
    public float oldPercent;
    public float newPercent;
    public float deltaPercent;
}

// Событие завершения фокуса
public class FocusCompletedEvent {
    public string skillId;
    public int minutes;
    public float skillGain;
    public int coinsReward;
    public int xpReward;
}

// Событие изменения параметров питомца
public class PetStatChangedEvent {
    public PetStatType stat; // Hunger, Energy, Mood, Health
    public int oldValue;
    public int newValue;
}
```

**SkillListItem — каждый элемент:**

- Название (до 20 символов, обрезается с "...")
- ProgressBar (заполнение от 0 до 100%)
- Текст процента (например "85%")
- Кнопка 🗑️ (активна ТОЛЬКО если percent == 0)
- Кнопка 🎯 (быстрый фокус — открывает FocusPanel с этим навыком)

### 5.3 Формула роста навыка

```csharp
float CalculateSkillGain(int minutes, int petLevel, int petMood) {
    float baseGain = (minutes / 5f) * GetPetLevelMultiplier(petLevel);
    float moodBonus = 0.5f + (petMood / 100f); // от 0.5 до 1.5
    float finalGain = baseGain * moodBonus;
    return Mathf.Min(finalGain, 100f - currentPercent); // не выше 100%
}

float GetPetLevelMultiplier(int level) {
    return 1.0f + (level - 1) * 0.02f; // уровень 1 → 1.0, уровень 10 → 1.18
}
```

**Примеры:**
- 15 минут, уровень 1, настроение 50 → (15/5)*1.0 = 3.0%, 3.0 * 1.0 = 3.0%
- 15 минут, уровень 5, настроение 100 → (15/5)*1.08 = 3.24%, 3.24 * 1.5 = 4.86%
- 60 минут, уровень 10, настроение 100 → (60/5)*1.18 = 14.16%, 14.16 * 1.5 = 21.24%

### 8.2 Техническая реализация таймера

```csharp
public class FocusTimer : MonoBehaviour {
    private float remainingSeconds;
    private bool isRunning;
    private Coroutine timerCoroutine;
    
    public void StartTimer(int minutes) {
        remainingSeconds = minutes * 60;
        isRunning = true;
        timerCoroutine = StartCoroutine(TimerCoroutine());
    }
    
    private IEnumerator TimerCoroutine() {
        float lastTime = Time.realtimeSinceStartup;
        while (isRunning && remainingSeconds > 0) {
            float now = Time.realtimeSinceStartup;
            float delta = now - lastTime;
            lastTime = now;
            remainingSeconds -= delta;
            UpdateUI(remainingSeconds);
            yield return null;
        }
        if (remainingSeconds <= 0) {
            OnTimerComplete();
        }
    }
    
    public void Pause() { isRunning = false; }
    public void Resume() { /* перезапуск корутины */ }
    public void CompleteEarly() { OnTimerComplete(); }
}
```

```csharp
public FocusReward CalculateReward(int minutes, Skill skill, PetStats pet) {

}
```

## 10. БОЕВАЯ СИСТЕМА (RPG LITE) — УМНЫЕ БОССЫ

### 10.1 Новая логика — сравнение паутин

Главная идея: У каждого босса есть своя «паутина навыков» — набор навыков, в которых он силён или слаб.

- Если твой навык выше, чем у босса → ты наносишь бонусный урон
- Если твой навык ниже, чем у босса → босс наносит дополнительный урон тебе
- Если навыков нет в паутине босса — нейтрально

### 10.2 Структура босса (расширенная)

```csharp
public class Boss {
    public string id;
    public string name;                    // "Математический дракон"
    public int maxHealth;
    public int currentHealth;
    public int baseDamage;                 // базовый урон босса (без учёта паутины)
```

### 10.4 Расчёт урона (новая формула)

```csharp
public class BattleCalculator {
    
    public (int petDamage, int bossDamage, string advice) CalculateDamage(
        PetStats pet, 
        List<Skill> playerSkills, 
        Boss boss
    ) {
        // Базовый урон питомца
        int basePetDamage = pet.level * 2 + Random.Range(1, 7);
        
        // Бонус/штраф от сравнения паутин
        float petDamageMultiplier = 1.0f;
        float bossDamageMultiplier = 1.0f;
        string advice = "";
        
        foreach (var bossSkill in boss.skillWeb) {
            string skillId = bossSkill.Key;
            float bossPercent = bossSkill.Value;
            
            var playerSkill = playerSkills.Find(s => s.id == skillId);
            float playerPercent = playerSkill?.percent ?? 0;
            
            if (playerPercent > bossPercent + 10) {
                // Игрок значительно сильнее → бонус к атаке
                petDamageMultiplier += 0.1f;
            } 
            else if (playerPercent < bossPercent - 10) {
                // Игрок слабее → босс наносит больше урона
                bossDamageMultiplier += 0.15f;
                advice = $"Босс сильнее в {playerSkill?.name ?? skillId}! Развивай этот навык!";
            }
        }
        
        int finalPetDamage = Mathf.RoundToInt(basePetDamage * petDamageMultiplier);
        int finalBossDamage = Mathf.RoundToInt(boss.baseDamage * bossDamageMultiplier);
        
        return (finalPetDamage, finalBossDamage, advice);
    }
}
```

### 10.5 Flow битвы (обновлённый)

```
1. Игрок нажимает [⚔️ Битва]
2. Выбирается случайный босс (из списка 5+)
3. Показывается экран битвы с паутиной босса и сравнением
4. Игрок нажимает [АТАКОВАТЬ]:
   - Проверка энергии (>=10)
   - Расчёт урона (с учётом сравнения паутин)
   - Уменьшение HP босса
   - Босс атакует в ответ
   - Снижение энергии на 10
5. Показывается совет: какой навык нужно развивать
6. Если HP питомца <= 0 → поражение (возврат с 0 наградой)
7. Если HP босса <= 0 → победа, награда
```

## 20. ОТВЕТЫ НА ЧАСТЫЕ ВОПРОСЫ РАЗРАБОТЧИКА

**Q: Как часто сохранять?**
A: После каждого действия: фокус, покупка, кормление, задание, битва, рутина. А также при паузе и каждые 60 секунд.

**Q: Как считать «пропущенный день»?**
A: Сравниваем lastActiveTime с текущим временем. Если разница > 24 часов — применяем штраф к голоду.

**Q: Можно ли добавить БОЛЬШЕ 12 навыков?**
A: ДА. НЕТ ОГРАНИЧЕНИЙ. Диаграмма показывает ТОП-12, список показывает все.

**Q: Что, если игрок хочет удалить навык с 80%?**
A: НЕЛЬЗЯ. Кнопка удаления неактивна. В будущем — «перековка» за монеты (v1.5).

**Q: Как работают умные боссы?**
A: У каждого босса своя паутина навыков. Сравнивается с паутиной игрока. Если твой навык ниже — получаешь больше урона. Если выше — наносишь больше.

**Q: Чем рутина отличается от навыкового задания?**
A: Рутина не требует фокуса и таймера. Игрок просто отмечает выполнение и получает награду. Подходит для простых привычек (вода, сон, душ).

**Q: Можно ли создать свою рутину?**
A: ДА. Игрок может создать любую рутину с названием и выбрать награду (монеты, опыт, энергия, настроение, прогресс навыка).

**Q: Как босс подсказывает, что развивать?**
A: В интерфейсе битвы отображается сравнение паутин. Если у босса навык выше, показывается совет: «Босс сильнее в Математике! Развивай её!».

**Q: Как реализована паутина сейчас?**
A: Своим UGUI-компонентом `RadarChartGraphic`, который рисует оси, кольца, полигон, outline и точки без внешнего chart-ассета.
